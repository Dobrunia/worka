# Архитектура приложения

## Процессная модель

```
┌─────────────────────────────────────────────────────────────┐
│                     npm run dev (shell)                     │
│                          PID: 12345                         │
└─────────────────────┬───────────────────────────────────────┘
                      │ запускает
                      ▼
┌─────────────────────────────────────────────────────────────┐
│                    tauri dev (Tauri CLI)                    │
│                          PID: 12346                         │
└──────────────┬────────────────────────────────┬─────────────┘
               │                                │
               │ запускает                      │ запускает
               ▼                                ▼
    ┌──────────────────────┐      ┌──────────────────────────┐
    │   npm run dev:vite   │      │   cargo run --bin worka  │
    │   (Vite dev server)  │      │   (Tauri Rust app)       │
    │   PID: 12347         │      │   PID: 12348             │
    │   Порт: 1420         │      │   + Windows tray         │
    └──────────────────────┘      │   + SQLite connection    │
                                  │   + Sampler thread       │
                                  └──────────────────────────┘
```

### Компоненты и их жизненный цикл

| Компонент | Процесс | Зависимости | Жизненный цикл |
|-----------|---------|-------------|----------------|
| **npm run dev** | Shell | — | Родительский процесс, закрывается по Ctrl+C |
| **tauri dev** | Tauri CLI | npm | Управляет Vite и Rust, закрывается по Ctrl+C |
| **Vite dev server** | Node.js | tauri dev | Frontend dev server, порт 1420 |
| **worka.exe** | Rust | tauri dev | Backend + UI окно + трей + сэмплер |

### Важные правила

1. **Vite может работать без Rust** — для разработки UI (`npm run dev:vite`)
2. **Rust НЕ должен работать без Vite** — будет белый экран
3. **При закрытии Ctrl+C** — все процессы должны закрыться
4. **При пересборке** — старый процесс закрывается, новый запускается

### Single-instance защита

- **Windows Mutex** — `Local\WorkaSingleInstanceMutex`
- **Второй процесс закрывается сразу** — `std::process::exit(0)`
- **Только одна иконка в трее** — гарантировано mutex

## Общая схема

```
[ Tray App ]
      │
      ├── Agent (сборщик активности)
      │       ├── Канал 1: Время
      │       ├── Канал 2: Ввод
      │       └── Канал 3: Метаданные
      │
      ├── Storage (SQLite)
      │
      └── Dashboard (Vue UI)
```

# Компоненты

## 1. Agent (фоновый сборщик)

Работает в фоне, пока приложение в трее.

### Канал 1 — Время
Сэмплинг каждые N секунд:
- активное приложение (exe / путь)
- статус idle (по порогу)
- (опционально) заголовок окна

Записывает `activity_sample`.

---

### Канал 2 — Ввод (только количество)

Через глобальные хуки Windows:
- счётчик нажатий клавиатуры
- счётчик кликов мыши
- (опционально) прокрутка

⚠️ Не сохраняется текст, только количество.

Агрегируется по времени (например по минутам) и записывается в `input_sample`.

---

### Канал 3 — Метаданные

- состояние паузы
- настройки интервала
- порог idle
- включён ли трекинг ввода
- включён ли трекинг заголовков окон

---

## 2. Storage (SQLite)

Отвечает за:
- миграции
- запись сэмплов
- агрегированные запросы
- получение данных для дашборда

- id

- display_name

## Таблица: activity_sample

Снимок состояния по времени.

- id
- timestamp
- app_id
- is_idle (bool)
- window_title (nullable)

---

## Таблица: input_sample

Агрегат по времени (например 1 минута).

- id
- timestamp_bucket
- keyboard_presses
- mouse_clicks

## Таблица: settings

- sample_interval_seconds
- idle_threshold_seconds
- track_window_titles (bool)
- track_input (bool)
- autostart (bool)
- paused (bool)

# Dashboard (Vue UI)

## Экран «Сегодня»
- KPI:
  - активное время
  - idle время
  - нажатия клавиш
  - клики мыши
- Топ приложений
- График интенсивности ввода

---

## Экран «Неделя»
- Активное/idle по дням
- Топ приложений за неделю
- Суммарный ввод за неделю

---

## Экран «Таймлайн»
- Лента дня (приложение / idle)
- (опционально) вторая дорожка интенсивности ввода

- Интервал опроса
- Порог idle
- Переключатели приватности
- Пауза / Продолжить
- Автозапуск

# Поведение приложения

## Single-instance режим
- **Только один экземпляр** — при запуске второго экземпляра открывается окно первого
- **Защита от дублирования** — используется Windows Mutex
- **Один сборщик** — трекинг выполняется в единственном экземпляре

## Запуск приложения
1. Проверка на единственный экземпляр (Mutex)
2. Если уже запущен → показать существующее окно
3. Если первый → создать tray, запустить Agent

## Работа с окном
- **Закрытие (крестик)** → скрыть окно в трей (не закрывать)
- **Клик по трей** → открыть меню (Открыть / Выйти)
- **Выход через трей** → остановить сэмплер, закрыть приложение

## Сбор данных
- Периодический сэмплер (каждые 10 секунд)
- Флаг остановки (`AtomicBool`) для корректного завершения
- Игнорирование `.db` файлов при пересборке (dev режим)

# Технологические детали

## Tauri плагины
- `tauri-plugin-opener` — открытие ссылок

## Windows API
- `GetForegroundWindow` — получение активного окна
- `GetWindowThreadProcessId` — получение процесса по окну
- `GetWindowTextW` — получение заголовка окна
- `GetLastInputInfo` — получение времени простоя
- `CreateMutexW` — создание mutex для single-instance

## Rust паттерны
- `Arc<AtomicBool>` — атомарный флаг остановки
- `Mutex<AppState>` — потокобезопасное состояние
- `std::thread::spawn` — фоновый сэмплер

- **Frontend:** 24 теста (Vitest)
- **Backend:** 9 тестов (Cargo)
